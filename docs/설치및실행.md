# 교회 재정 관리 — 설치 및 실행 방법

웹 브라우저 UI + **SQLite DB**를 사용하는 재정 관리 프로그램입니다. PC에 **Node.js**가 필요합니다.

---

## 1. 필요 환경

| 항목 | 권장 |
|------|------|
| **운영체제** | Windows 10 이상 |
| **Node.js** | 18.x 이상 (LTS 권장) |
| **npm** | Node.js 설치 시 함께 제공 |
| **브라우저** | Chrome, Edge, Firefox 등 최신 버전 |

```bash
node -v
npm -v
```

---

## 2. 프로그램 준비 (최초 1회)

```bash
cd d:\재정프로그램
npm install
```

---

## 3. 실행 방법 (일반 사용)

### 방법 A — 배치 파일 (권장, Windows)

프로젝트 폴더에서 **`실행.bat`** 을 더블클릭합니다.

- API·웹 서버가 시작되고, 잠시 후 브라우저가 `http://localhost:5173` 으로 열립니다.
- 종료할 때는 검은 창(터미널)에서 `Ctrl + C` 를 누른 뒤 창을 닫습니다.

### 방법 B — 명령줄

`npm run dev` 한 번으로 **API 서버**와 **웹 화면**이 함께 실행됩니다.

```bash
cd d:\재정프로그램
npm run dev
```

| 구분 | 주소 |
|------|------|
| **웹 화면** | http://localhost:5173 |
| **API 서버** | http://localhost:3001 |

1. Chrome 또는 Edge에서 **http://localhost:5173** 접속
2. 상단에 **「SQLite 저장됨」** 이 보이면 DB 연결 정상

### 종료

터미널에서 `Ctrl + C` — API·웹 서버 모두 종료됩니다.

### 데이터 저장 위치

- **SQLite 파일:** `d:\재정프로그램\data\finance.db`
- 서버를 껐다 켜도 **DB 파일에 데이터가 유지**됩니다.
- 다른 PC로 옮길 때는 `data\finance.db` 파일과 프로그램을 함께 복사하거나, **백업** 메뉴의 JSON Export/Import를 사용합니다.

### API만 따로 실행 (선택)

```bash
npm run dev:server   # API만
npm run dev:client   # 웹만 (다른 터미널)
```

---

## 4. 빌드 후 실행

```bash
npm run build
npm run preview
```

`preview` 실행 시에도 API 서버가 함께 뜹니다. 브라우저 주소는 터미널 안내를 따릅니다.

---

## 5. 메뉴 안내

| 메뉴 | 용도 |
|------|------|
| **월간 실적** | 해당 월 일요일 기준 주차별 입력 |
| **연간 예산** | 연도별 예산·예산 대비 |
| **기간 통계** | 월별(주별) / 상반기(1~6월 월별) / 연간(1~12월 월별) |
| **계정 관리** | 계정 CRUD, **계정 한번에 여러개 입력하기** |
| **백업** | JSON 전체 백업·복원 |

## 6. 자주 묻는 문제

### `NODE_MODULE_VERSION` / `better_sqlite3.node` 오류

다른 PC에서 `node_modules` 폴더만 복사했거나, Node.js 버전이 바뀌면 발생합니다.

**해결 (해당 PC에서):**

1. **`수리-설치.bat`** 실행 (권장) — `node_modules` 삭제 후 재설치·빌드  
2. 또는 명령 프롬프트에서:
   ```bash
   cd 프로젝트폴더
   npm install
   npm rebuild better-sqlite3
   ```
3. 그다음 **`실행.bat`** 실행

**주의:** 프로젝트를 옮길 때는 `node_modules` 는 복사하지 말고, 소스만 복사한 뒤 위 절차를 실행하세요.  
Node.js **24** 등 최신 버전에서 빌드가 실패하면 [Node.js LTS 22.x](https://nodejs.org/) 설치를 권장합니다.

### 「서버에 연결할 수 없습니다」

- `실행.bat` 또는 `npm run dev` 로 실행했는지 확인 (API + 웹 동시 실행)
- 위 SQLite 오류로 API가 죽으면 `ECONNREFUSED` 가 함께 보일 수 있음
- 방화벽이 3001 포트를 막지 않는지 확인

### 계정 이름 변경 창이 안 뜸

- **일반 브라우저**에서 접속 (`prompt` 사용). IDE 내장 미리보기는 동작하지 않을 수 있음.

### 이전 localStorage 데이터

- SQLite 전환 이후 **예전 브라우저 데이터는 자동 이전되지 않음**. 백업 JSON이 있으면 Import.
