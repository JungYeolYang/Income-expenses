# 교회 재정 관리 프로그램 — 설계 문서

## 1. 개요

### 1.1 목적

수입·지출을 **계정별**로 **매주** 입력·저장하고, 누적 데이터를 바탕으로 **엑셀 파일**로보내 인쇄·보관·공유할 수 있는 **웹 기반** 재정 관리 도구를 만든다.

### 1.2 대상 사용자

교회(또는 유사 단체) 재정 담당자 — **웹 브라우저** + 로컬 **API 서버**로 사용한다.

### 1.3 범위 요약

| 구분 | 내용 |
|------|------|
| 입력 단위 | 주(Week) 단위 — **해당 월의 일요일** 기준 주차, **해당 월 전체 주**를 한 화면에서 입력·편집 |
| 월간 화면 | 월 내 주별 금액 + **해당 월 합산 통계** + **연간 예산(계정별)** 동시 표시 |
| 분류 | 수입 / 지출 |
| 계정 구조 | 대분류 → 세부 계정 (사용자 정의 가능) |
| 연간 예산 | 계정별 연간 예산 입력·저장 (별도 화면) |
| 기간 통계 | **월별·상반기·연간** 3종 (§2.3) |
| 저장 | **SQLite** (`data/finance.db`) — Node API 서버 경유 |
| 계정 관리 | 추가·삭제·변경, **계정 한번에 여러개 입력하기**, Import/Export |
| 엑셀 | 통계 3종 + 월간 실적 — 화면과 동일 표, 수입·지출 시트 분리 |

## 2. 기능 요구사항

### 2.1 1단계 — 주간 입력·저장 (월 단위 화면)

- 실적 입력의 **기본 단위는 주(Week)** 이다. 매주 수입·지출 계정별 금액을 기록한다.
- 입력 화면은 **한 주만** 보여 주는 방식이 아니라, **선택한 달에 속한 모든 주**를 **한 화면**에 배치한다.
  - 예: 행 = 계정(또는 대분류·세부 계정), 열 = 1주차·2주차·…·n주차 + **월 합계** 열
  - 해당 월의 각 주 셀에서 **직접 입력·수정** 가능
  - 화면 하단 또는 우측 등 **월 합산 통계**(수입 합계, 지출 합계, 계정별 월 합계 등)를 **같은 화면**에서 즉시 확인
- 입력한 데이터는 **영구 저장**되며, 브라우저를 닫았다가 다시 열어도 유지되어야 한다.
- 월을 변경하면 해당 월의 주별 데이터·월 합계가 함께 갱신되어 표시된다.

#### 2.1.1 월간 화면과 연간 예산 표시

- **월 합산 결과**가 보이는 영역 **부근**(같은 화면 내 인접 블록)에, **해당 연도**에 집행하기로 한 **연간 예산**을 함께 표시한다.
  - 계정(또는 세부 계정)별 **연간 예산액**
  - (선택) 월 누계 실적 vs 연간 예산, 잔여 예산 등 파생 지표
- 연간 예산 값은 **§2.5 연간 예산 입력** 화면에서 등록·수정한 데이터를 참조한다.

### 2.2 2단계 — 계정 관리

- 계정 **추가**, **삭제**, **이름·분류 변경**을 UI에서 수행할 수 있어야 한다.
- **계정 한번에 여러개 입력하기**: 여러 계정을 한 번에 등록할 수 있어야 한다.
- **Export / Import**: 계정 목록(구조)을 파일로보내고, 다른 PC·백업·초기 설정 시 다시 가져올 수 있어야 한다.
- 아래 **§3 기본 계정 목록**은 **초기 템플릿**이며, 운영 중 변경 가능하다.

### 2.3 2단계 — 기간별 통계 (월별·상반기·연간)

통계 화면에서 선택하는 기간은 **3가지만** 제공한다.

| 종류 | 범위 | 열(집계 단위) | 비고 |
|------|------|----------------|------|
| **월별** | 선택한 **1개월** | 해당 월 **주별**(일요일 기준) | 월간 실적과 동일 주차 |
| **상반기** | **1월~6월** | **월별** 집계(1월…6월) | 주별 열 없음 |
| **연간** | **1월~12월** | **월별** 집계(1월…12월) | 주별 열 없음 |

공통 표시 열(계정별):

| 표시 항목 | 설명 |
|-----------|------|
| **기간 열** | 위 표의 주별 또는 월별 실적 |
| **기간합계** | 선택 기간 실적 합계 |
| **연간 예산** | 해당 연도 연간 예산 (안분 없음) |
| **연 누계** | 해당 연 1월~기간 말까지 누적 실적 |
| **예산 대비** | 차이(금액·%), 집행률(%) — §5.2 |

- **분기·하반기 단독** 통계는 제공하지 않는다.
- 수입·지출 표를 분리하여 표시한다.

### 2.4 엑셀 Export

- **월간 실적** 화면: 해당 월 주별 표 (수입·지출 시트)
- **통계** 화면: 선택한 통계 종류(월별·상반기·연간)와 **동일한 열 구조**로 Export
- 스타일: 헤더·테두리·수입/지출 색 등 화면과 맞춘 서식 (§7)

### 2.5 연간 예산 화면 (입력 + 예산 대비 실적)

- **연도**를 선택하는 **전용 화면**에서 다음을 한 표로 다룬다.
  - **연간 예산 입력·수정** (수입·지출, 계정별)
  - **예산 대비 실적** 비교 (계정별, **금액·퍼센트** 모두 표시)
- 계정 구조(대분류·세부 계정)는 실적 입력과 **동일한 계정 트리**를 사용한다.
- 계정이 추가·변경되면 예산 화면에도 반영된다(신규 계정은 예산 0 또는 미입력 상태).

#### 2.5.1 표시·계산 항목 (계정별)

| 열 | 설명 | 계산 (연간 예산 = B, 연 누계 실적 = A) |
|----|------|----------------------------------------|
| **연간 예산** | 사용자 입력 | B |
| **연 누계 실적** | 해당 연도 주간 입력 합계 | A (1월~**마지막 입력 주**까지, §5.2) |
| **차이(금액)** | 예산 대비 실적 격차 | A − B |
| **차이(%)** | 예산 대비 격차 비율 | B ≠ 0 일 때 (A − B) ÷ B × 100 |
| **집행률(%)** | 예산 대비 실적 비중 | B ≠ 0 일 때 A ÷ B × 100 |

- **차이(금액)·차이(%)** 를 **반드시** 계정별로 표시한다.
- **집행률(%)** 은 차이(%)와 중복 정보이나, 가독성을 위해 함께 두거나 차이(%)만으로 대체 가능 — **1차 구현 시 차이(금액)·차이(%) + 집행률(%)** 세 열 권장.
- 연간 예산이 **0**인 계정: 차이(%), 집행률(%)는 **「-」** 또는 빈칸 (0으로 나누기 방지).
- **대분류 소계·수입/지출 총계** 행에도 동일 규칙으로 합산·재계산한다.
- (선택) 수입·지출에 따라 차이 부호 해석 안내(툴팁): 지출은 실적이 예산을 넘으면 차이(+) = 초과 집행 등.

#### 2.5.2 실적 기준일

- 연간 예산 화면의 실적은 **해당 연도 전체 연 누계**를 사용한다.
- 「오늘」이 아니라 **입력된 주간 데이터가 있는 마지막 주**까지 누적하는 방식으로도 일관되게 처리할 수 있다(§5.2와 동일).

#### 2.5.3 참조·저장

- 연간 예산은 다음에서 참조된다.
  - **월간 입력 화면**: 해당 연도 예산 표시(§2.1.1)
  - **월별·상반기·연간 통계**: 예산 대비 결산(§2.3)
  - **엑셀 Export**
- 연도별로 예산 데이터를 분리 저장한다(예: 2025년 예산, 2026년 예산).

### 2.6 데이터 저장·백업 (SQLite)

- **SQLite** 파일(`data/finance.db`)에 계정·주간 실적·연간 예산을 저장한다.
- **Node.js API 서버**가 DB를 읽고 쓴다. 브라우저는 `localStorage`가 아닌 **REST API**를 사용한다.
- 서버를 종료해도 DB 파일에 데이터가 유지된다.
- **백업** 메뉴: 전체 데이터 JSON Export / Import (DB 덮어쓰기).
- 계정만 JSON Import/Export 가능(기존과 동일).

## 4. 기술 방향

### 4.1 실행 환경

- **웹 브라우저**에서 동작하는 SPA 또는 정적 웹 앱
- 별도 백엔드·DB 서버 **불필요**

### 4.2 데이터 저장

| 저장 대상 | 방안 (예시) |
|-----------|-------------|
| 계정 구조 | IndexedDB / localStorage + JSON Export |
| 주간 실적 | 동일 저장소 또는 JSON/파일 백업 |
| 연간 예산 (연도·계정별) | 동일 저장소 + Export/Import |
| 설정 | 로컬 저장 |

- 데이터 손실 방지를 위해 **정기 Export(백업)** 사용을 권장한다.

### 4.3 엑셀 처리

- 클라이언트에서 `.xlsx` 생성 (예: SheetJS, ExcelJS 등 라이브러리)
- A4 인쇄를 고려한 **열 너비, 머리글, 합계 행, 인쇄 영역** 설정

### 4.4 계정 Import/Export 형식 (안)

- **JSON** 또는 **CSV** 권장
  - JSON: 대분류·세부 계정·수입/지출 구분을 트리로 표현하기 용이
  - CSV: 벌크 편집·엑셀에서 직접 수정 후 Import에 용이
- Export 파일 예시 구조 (개념):

```json
{
  "version": 1,
  "accounts": {
    "income": [
      {
        "category": "일반헌금",
        "items": ["주일헌금", "십일조"]
      }
    ],
    "expense": [
      {
        "category": "적립금",
        "items": ["사례비", "상여금"]
      }
    ]
  }
}
```

## 5. 개념적 데이터 모델

```
AccountCategory (대분류)
  ├── type: income | expense
  ├── name: string
  └── items: AccountItem[]

AccountItem (세부 계정)
  ├── id: string (고유 ID)
  └── name: string

WeeklyEntry (주간 실적)
  ├── year, month: number
  ├── sundayDay: number           // 해당 월 일요일 날짜(일)
  └── amounts: { [accountItemId]: number }
  // 저장 키: `${year}-${month}-${sundayDay}`

AnnualBudget (연간 예산)
  ├── year: number
  └── amounts: { [accountItemId]: number }   // 연간 예산액

Settings
  ├── defaultCurrency: KRW
  └── (기타 UI·표시 설정)
```

### 5.1 집계 규칙

| 집계 대상 | 규칙 |
|-----------|------|
| **월 합계** | 해당 월 모든 주간 실적 합산 |
| **월별 통계** | 한 달, 열 = **주별**(일요일) |
| **상반기 통계** | 1~6월, 열 = **월별** |
| **연간 통계** | 1~12월, 열 = **월별** |
| **기간합계** | 해당 통계 기간 실적 합 |
| **연 누계** | 해당 연 1월~통계 기간 말 누적 |

### 5.2 예산 표기·예산 대비 결산 규칙 (확정)

- **예산 데이터**: `AnnualBudget`에 **연도·계정별 연간 예산**만 저장한다. 분기별·월별·반기별 예산 필드는 **두지 않는다**.
- **예산 표기** (월간 입력, 통계, 엑셀):
  - 모든 화면에서 **연간 예산** 열만 표시한다.
  - 모든 통계에서 예산 숫자는 **그 해 연간 예산과 동일** (안분 없음).
- **결산**:
  - **기간 결산** = 선택 통계 기간의 실적 합.
  - **연 누계 실적** = 해당 **연도 1월 1일(또는 1월 첫 주)** 부터 **선택 기간 말**까지의 주간 실적 누적 합.
- **예산 대비** (비교용 파생 값, 계정별):
  - **차이(금액)** = 연 누계 실적 − 연간 예산
  - **차이(%)** = (연 누계 실적 − 연간 예산) ÷ 연간 예산 × 100 (예산 ≠ 0)
  - **집행률(%)** = 연 누계 실적 ÷ 연간 예산 × 100 (예산 ≠ 0)
  - **연간 예산 화면**(§2.5)에서는 위 항목을 **금액·% 모두** 표 컬럼으로 표시한다.
  - 월별·상반기·연간 통계 화면에서 **동일 공식** 적용.

## 6. 화면 구성 (안)

### 6.1 화면 목록

| 화면 | 설명 |
|------|------|
| **월간 실적 입력** | 연·월 선택 → 해당 월 **전체 주** 열 + 계정별 입력, **월 합계**, **해당 연도 연간 예산** 인접 표시 |
| **연간 예산** | 연도 선택 → 계정별 **예산 입력** + **연 누계 실적** + **차이(금액·%)** + 집행률(%) |
| **기간별 통계** | **월별**(주별) / **상반기**(월별 1~6) / **연간**(월별 1~12) |
| 계정 관리 | CRUD, **계정 한번에 여러개 입력하기**, Import/Export |
| 보고서 (향후) | 기간·단위 선택 후 엑셀 Download |
| 설정 (선택) | 백업, 초기 템플릿 복원, 주 기준일 등 |

### 6.2 월간 실적 입력 화면 (와이어프레임 개념)

```
[ ◀ 2026년 3월 ▶ ]

┌─────────────────────────────────────────────────────────────┐
│  수입 │ 계정      │ 1주 │ 2주 │ 3주 │ 4주 │ 5주 │ 월합계 │ 연간예산 │
│  ...  │ 주일헌금  │ ... │ ... │ ... │ ... │     │  xxx   │   xxx    │
└─────────────────────────────────────────────────────────────┘
┌─────────────────────────────────────────────────────────────┐
│  지출 │ ...       │ ... │ ... │ ... │ ... │     │  xxx   │   xxx    │
└─────────────────────────────────────────────────────────────┘
[ 월 수입 합계 │ 월 지출 합계 │ (선택) 월 순잉여 ]
```

- 주 개수는 **달마다 4~5주** 가변.
- **연간 예산** 열은 읽기 전용(예산 화면에서 편집).

### 6.3 기간별 통계 화면 (와이어프레임 개념)

```
[ 2026년 3월 │ ● 월별 ○ 상반기 ○ 연간 ]

월별:  │ 1주(3/2) │ 2주(3/9) │ ... │ 월합계 │ 연간예산 │ 연누계 │ 차이 │ ...

상반기: │ 1월 │ 2월 │ ... │ 6월 │ 상반기합계 │ 연간예산 │ ...

연간:   │ 1월 │ ... │ 12월 │ 연간합계 │ ...
```

### 6.4 연간 예산 화면 (와이어프레임 개념)

```
[ ◀ 2026년 ▶ ]     실적 기준: 2026년 연 누계 (1월 ~ 5월 4주차)

┌────────────────────────────────────────────────────────────────────────┐
│ 수입 │ 계정     │ 연간예산 │ 연누계실적 │ 차이(금액) │ 차이(%) │ 집행률(%) │
│      │ 주일헌금 │  [입력]  │   xxx      │   xxx      │  xx.x%  │  xx.x%    │
│      │ (소계)   │   xxx    │   xxx      │   xxx      │  xx.x%  │  xx.x%    │
├────────────────────────────────────────────────────────────────────────┤
│ 지출 │ 사례비   │  [입력]  │   xxx      │   xxx      │  xx.x%  │  xx.x%    │
└────────────────────────────────────────────────────────────────────────┘
```

- **연간예산** 열만 편집 가능, 실적·차이·% 열은 **자동 계산**(읽기 전용).
- 주간 실적이 갱신되면 이 화면 수치도 **즉시 반영**된다.

## 7. 엑셀 출력 레이아웃 (A4) — 가이드

향후 보고서 Export 시 다음을 준수한다.

1. **용지:** A4, 세로(또는 표 폭에 따라 가로) 방향
2. **상단:** 교회명(또는 제목), 보고 기간, 수입/지출 구분
3. **본문:** 대분류별 소계, 세부 계정별 금액, 우측 정렬 숫자
4. **하단:** 수입 합계, 지출 합계, (필요 시) 순잉여
5. **연간 예산·결산:** 보고서에 **연간 예산**, 기간/연 누계 실적, 잔여·집행률 열 포함 (기간 안분 예산 없음)
6. **인쇄:** 인쇄 제목 행, 페이지 번호, 적절한 `print area` 설정

## 8. 미확정 사항 (추후 결정)

| 항목 | 설명 |
|------|------|
| 주(Week) 정의 | 캘린더 주(일~토) vs 회계 주 vs “해당 주 일요일” 기준일 |
| 금액 단위 | 원 정수, 소수점, 음수(환불·조정) 허용 여부 |
| 주간 실적 Import/Export | 1차 범위에 포함할지, 계정만 1차로 할지 |
| 다중 사용자 | 단일 PC·단일 담당자 가정 vs 파일 공유 |
| 인증·권한 | 로컬 전용이면 불필요 |
